JotaComplaint

A modern, full-stack Municipal Complaint Management System that empowers citizens to report issues, enables staff to resolve them, and gives administrators real-time visibility.

Next.js 16 TypeScript Prisma ORM PostgreSQL Auth.js Tailwind CSS Leaflet Maps ImageKit CDN
Abstract

01 What is JotaComplaint?

🎯
The Problem
Municipal complaints are lodged via phone calls or physical visits — with zero transparency, no tracking, and no accountability. Complaints go unresolved for weeks.
💡
Our Solution
A three-portal web platform (Citizen, Staff, Admin) built with the T3 Stack — enabling photo uploads, GPS-tagged locations, real-time status tracking, and analytics dashboards.
📌
Aim
To design and develop a scalable, role-based web application that digitizes the municipal complaint management process — improving transparency, accountability, and resolution time for civic issues.
Objectives

02 Our Key Objectives

  • 1 Citizen Reporting — Submit complaints with photos, GPS location, and categorization
  • 2 Automated Routing — Assign complaints to relevant departments by category
  • 3 Real-time Tracking — Citizens can monitor live status updates at every stage
  • 4 Staff Portal — Dedicated hub for assigned complaints, internal notes, and field navigation
  • 5 Admin Dashboard — Analytics, charts, user/department management
  • 6 Audit Trail — Immutable activity logs for every action on a complaint
  • 7 Notification System — In-app alerts at each lifecycle stage
  • 8 Secure by Design — RBAC, bcrypt hashing, DB sessions, Zod validation

🗄️ Architecture
Monolithic full-stack — Server Components + Server Actions (no REST API needed)
🔐 Auth Strategy
Database sessions via Auth.js + Prisma adapter. Middleware guards every route by role.
📊 Data Layer
8 models, 7 enums, indexed FKs, soft deletes, cascade integrity via Prisma ORM.
